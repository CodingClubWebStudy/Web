HTML4까지는 <div>태그를 이용해 화면을 분할하거나 그 안에 제목을 넣었지만, HTML5에서는 모든 태그에 자신의 역할이 생겨났다.
이런 태그들을 시맨틱 태그(semantic tag)라고 한다.


  태그만 보고도 쉽게 페이지 구조를 이해할 수 있다. 사람들이 소스를 쉽게 이해할 수 있도록 태그를 의미가 통하게 시맨틱 태그로 바뀌었다.
<div> 태그는 웹 문서의 이미지나 내용 등을 담아놓는다.
보통 사이트의 맨 윗부분은 사이트 로고 이미지와 제목등이 포함된 '헤더(header)'로 사용된다.
그래서 헤더부분의 내용을 <div> 로 묶고 그 이름을 id="header"라고 지정하는것이 일반적이다.

하지만! 개발자에 따라 id="top" 또는 다른 이름으로 지정 가능하다.
(이렇게 되면 담당자가 바뀌거나 수정시에도 보기가 어렵고 수정하기 힘들다.)


simantic - 의미가 통하는 (형용사)

HTML4에서는 웹 문서에 메뉴를 넣을 때, 본문 글을 넣을때 모두 <div>태그를 사용.
HTML5에서는 메뉴를 넣을 때는 <nav> 
                         본문의 글을 넣을 때 <article>


HTML5를 이용하여 웹 표준 문서를 만들려면 시맨틱 태그를 사용해야 한다.
<header> 태그 - 머리말 (제목)
<header> </header>

<head> 와 <header>태그는 다르다.
<head> : 문서에서 단 한번만 사용가능. <title>, <meta>태그 같은 웹 문서 정보를 넣는다. 실제 웹 화면에는 보이지 않는 내용.
<header> : <body>태그 안에 사용하는 태그로, 한 문서에서 여러 번 사용 가능. 사이트 로고나 메뉴 등 실제 문서 내용 입력.
